# 页面风格规则

本文用于固化当前项目已经确定下来的页面风格。后续新增页面、调整页面，默认都要遵循本文，不再各页单独发散。

适用范围：
- 左侧导航壳层
- 列表页
- 新增页 / 详情页 / 配置页
- 仪表盘类页面

当前风格基线来源：
- `resources/css/prototype.tokens.css`
- `resources/css/prototype.shell.css`
- `resources/css/prototype.table.css`
- `resources/css/prototype.dashboard.css`
- `resources/css/pages/offline-analysis.css`
- `resources/css/pages/offline-task-create.css`
- `resources/css/pages/stream-analysis.css`

## 1. 风格定位

当前项目的页面风格定位为：浅色、克制、偏业务后台、强调信息清晰度。

核心关键词：
- 浅底色
- 白色内容卡片
- 蓝色作为唯一主强调色
- 灰色作为中性辅助色
- 红色仅用于危险操作和异常提示
- 低装饰、低阴影、低渐变

明确要求：
- 不做大面积炫光渐变背景
- 不做深色后台风格
- 不做紫色系主视觉
- 不做花哨动效
- 不做过度卡片堆叠

说明：
- 当前项目早期部分页面存在较重的渐变和阴影，后续统一以已经收敛后的“离线视图分析 / 离线新建任务 / 当前视频流分析”这套风格为准
- 新页面优先继承这套已收敛风格，不再回退到旧风格

## 2. 全局布局规则

### 2.1 页面骨架

统一使用后台壳层布局：
- 左侧导航
- 右侧主内容区
- 内容区顶部为白色顶栏
- 顶栏下方为页面主体内容

基础规则：
- 整体背景使用浅灰蓝底色
- 内容区与导航区分层明确
- 页面内容按模块纵向排列
- 模块之间使用稳定间距，不靠大色块区分层级

### 2.2 导航栏

左侧导航使用当前项目现有分组导航风格：
- 侧栏背景：白色
- 展开宽度：`252px`
- 收起宽度：`68px`
- 分组头：浅灰底、圆角矩形
- 二级菜单：白底列表
- 当前选中项：整条铺满的蓝色高亮底

交互规则：
- 允许一级分组展开/收起
- 当前项高亮必须铺满整行，不能只高亮文字
- 二级菜单 hover 可有轻微位移和浅蓝底

层级规则：
- 只保留“一级分组 + 二级菜单”结构
- 不再新增三级导航

### 2.3 顶栏

顶栏统一样式：
- 白底
- 细边框
- 圆角 `12px`
- 高度视觉上约 `56px`
- 左侧为折叠按钮和面包屑
- 右侧只保留必要操作

当前项目约束：
- 顶栏不再放全局搜索框，除非后续明确提出需求
- 页面标题以面包屑表达为主，内容区域内不重复堆一个大标题

## 3. 颜色规则

### 3.1 基础色板

统一优先使用以下颜色体系：

```css
--bg: #f3f6fb;
--sidebar-bg: #f8fafc;
--panel: #ffffff;
--panel-muted: #f7faff;
--line: #dbe4ef;
--line-strong: #c7d4e4;
--text: #273446;
--muted: #6f8095;
--accent: #2d83f5;
--danger: #ef7270;
--warning: #f2a85f;
--success: #56bf94;
```

页面局部允许的直接用色：
- 主按钮蓝：`#2d83f5`
- 主按钮 hover：`#225fdf`
- 次级操作灰：`#96a0af`
- 危险操作红：`#ff3b30`
- 文本链接蓝：`#2b64eb`
- 文本链接 hover：`#1d4fd3`
- 删除类文字链接红：`#ff4d4f`
- 删除类文字链接 hover：`#e23235`

### 3.2 用色原则

规则如下：
- 蓝色只用于“主要动作、当前状态、重点数据”
- 灰色用于“中性动作、辅助信息、次级标签”
- 红色只用于“删除、危险、失败、异常”
- 橙色只用于“告警、待处理、风险”
- 绿色只用于“成功、启用、正常”

禁止项：
- 同一页面内并行使用多套主色
- 出现紫色主题按钮或紫色主视觉
- 大面积彩色背景板

## 4. 字体与文字层级

统一使用项目现有字体体系：
- `Source Sans Pro`

推荐字号层级：
- 面包屑当前页：`18px / 700`
- 面包屑分组：`16px`
- 面板标题：`16px ~ 18px / 700`
- 表头：`12px / 700`
- 正文：`13px`
- 辅助文案：`12px`

文字风格要求：
- 标题偏稳重，不使用夸张字重差
- 正文优先保证清晰易读
- 辅助文案统一使用偏灰色
- 空状态文案必须完整可读，不能留空

## 5. 面板与容器规则

### 5.1 通用面板

所有业务内容模块优先放入白色面板中：
- 背景：白色
- 边框：浅灰蓝描边
- 圆角：`10px ~ 14px`
- 阴影：极轻，或直接无阴影

当前默认倾向：
- 列表页、创建页优先使用“弱阴影或无阴影”
- 不再使用厚重投影

### 5.2 内容密度

统一原则：
- 页面以“清晰、可扫读”为第一优先级
- 控件高度统一
- 内边距稳定
- 不用额外色块制造层级

推荐节奏：
- 主内容区模块间距：`16px`
- 面板内常规 padding：`16px ~ 20px`
- 工具栏与表格间距：`20px`

## 6. 列表页规则

列表页是当前项目的主模板，后续新页面优先对齐这一套。

### 6.1 工具栏

列表页工具栏统一规则：
- 不再额外套一层筛选卡片
- 筛选控件直接放在列表面板顶部
- 操作按钮和筛选条件分成左右两组

推荐布局：
- 左侧：主操作按钮组
- 右侧：条件筛选区

筛选区顺序：
- 先状态类下拉
- 再类型类下拉
- 再关键词搜索框
- 再 `查询`
- 最后 `重置`

说明：
- 如果页面业务上必须“先搜索再筛选”，需要明确说明，默认不这么做
- 任务类页面统一向“离线视图分析”看齐

### 6.2 筛选控件

筛选区输入控件统一要求：
- 高度：`40px`
- 圆角：`6px`
- 背景：白色
- 边框：浅灰蓝
- 不加厚阴影

建议宽度：
- 普通下拉：`142px`
- 搜索框：`220px`

### 6.3 操作按钮

列表页工具栏按钮规则：
- 高度：`40px`
- 最小宽度：`98px`
- 圆角：`4px`
- 字重：`700`

颜色规则：
- 主按钮：蓝底白字
- 次级批量操作：灰底白字
- 危险操作：红底白字
- 查询按钮：沿用主按钮样式
- 重置按钮：白底描边，深灰文字

特殊说明：
- 只有左侧主操作按钮组可以做“纯色实底 + 无边框”
- 筛选区中的 `重置` 必须保留白底描边样式，不能和左侧实色按钮混淆

### 6.4 表格

表格统一规则：
- 表格容器白底
- 外层圆角：`10px`
- 表头底色：`#f7faff`
- 单元格 padding：`12px 14px`
- hover 行背景：`#f9fbff`
- 线条颜色柔和，避免过深

表头风格：
- `12px`
- 字重 `700`
- 颜色偏蓝灰

正文风格：
- `13px`
- 深灰蓝色

### 6.5 状态标签

状态标签统一要求：
- 最小宽度：约 `72px`
- 圆角：`4px`
- 中等字重
- 不再使用过度圆润的胶囊样式作为默认风格

状态语义：
- 成功：浅绿底 + 绿色文字
- 警告：浅橙底 + 橙色文字
- 失败/异常：浅红底 + 红色文字
- 空闲/未开始：浅灰蓝底 + 蓝灰文字

### 6.6 行内操作

表格行内操作统一改为文字链接风格：
- 普通动作：蓝色文字链接
- hover：更深的蓝色
- 删除：红色文字链接

不再推荐：
- 行内操作做成一排圆角小按钮

### 6.7 分页

分页统一规则：
- 高度：`32px ~ 34px`
- 圆角：`8px`
- 白底描边
- 当前页：蓝底白字
- 不使用重阴影

## 7. 创建页 / 详情页规则

创建页和详情页统一参考“离线新建任务”这套形式。

### 7.1 布局

推荐结构：
- 页面主体仍放在白色面板里
- 面板内部按 section 分段
- 标签列和内容列清晰分开

推荐表单行：
- 左列标签宽度：约 `76px`
- 右列内容自适应
- 行间距：`16px ~ 18px`

### 7.2 表单控件与选择器

统一要求：
- 单选、标签、chip、下拉、输入框使用同一套浅蓝灰配色
- 激活态只使用蓝色，不增加第二套强调色
- 上传区用浅边框虚线框表达，不做重色块

### 7.3 分段区域

创建页中的“算法区 / 上传区 / 配置区”规则：
- 独立白色区域
- 边框浅、圆角中等
- 标题清晰
- 说明文字使用 `12px` 灰色

### 7.4 底部操作区

底部操作统一：
- 按钮居中排布
- 间距稳定
- 主次关系明确

## 8. 仪表盘类页面规则

仪表盘页允许比列表页稍强一点的视觉层次，但仍然不能跳出当前项目整体风格。

允许保留的表现：
- 数据卡片
- 图表
- 统计块
- 快捷入口

仍需遵守：
- 整体底色浅
- 卡片白底
- 主色依旧是蓝色
- 彩色只出现在指标图标、数值状态、图表曲线里
- 不允许整页做大面积强渐变

当前建议：
- metrics 卡片可以保留轻渐变图标
- 但页面外层容器、面板主体、工具栏仍然保持简洁

## 9. 文案与信息呈现规则

页面文案统一要求：
- 简短
- 业务化
- 不说“示例、测试、占位”这类弱化语气，除非该区域明确是原型说明

标题规则：
- 一个页面只保留一个主要标题表达源
- 当前项目以顶栏面包屑为主
- 列表面板内部默认不再重复放大标题

说明文案规则：
- 能省则省
- 只在需要解释字段、上传规则、异常边界时出现

## 10. 风格落地规则

后续新增页面时，按下面顺序执行：

1. 先判断页面属于列表页、创建页、详情页还是仪表盘页
2. 优先复用已有共享样式，不先写新的页面视觉体系
3. 页面局部差异放到 `resources/css/pages/*.css`
4. 如果同类样式会被两个及以上页面复用，再上收进共享样式文件
5. 如果需要新增颜色，先补 token，再使用

默认参考页：
- 列表页：`离线视图分析`
- 创建页：`离线新建任务`
- 任务列表页变体：`视频流分析`

## 11. 明确禁止项

后续开发中，默认禁止以下做法：
- 新页面单独定义一套主色
- 用紫色、黑金、深色科技风替换当前浅色后台风格
- 大面积使用重阴影
- 筛选区额外再包一层显眼卡片
- 在顶栏重新加回全局搜索框
- 行内操作继续做成胶囊小按钮
- 同类页面出现不同圆角体系
- 同类页面出现不同输入框高度
- 同类页面出现不同按钮高度

## 12. 页面开发检查清单

后续每次新增页面、重做页面、补页面时，默认都按下面清单自查一次。

### 12.1 通用检查

- 页面是否仍然使用当前后台壳层，而不是另起一套布局
- 左侧导航、顶部栏、内容区之间的关系是否与现有页面一致
- 页面是否仍然以浅色后台风格为主，没有引入新的主题风格
- 是否优先复用了共享样式，而不是先写一套新的局部体系
- 页面标题是否通过顶栏面包屑表达，没有在内容区重复堆叠大标题

### 12.2 颜色检查

- 主色是否仍然只使用蓝色体系
- 灰色是否只用于中性动作和辅助信息
- 红色是否只用于删除、危险、失败、异常
- 是否出现紫色、深色科技风、黑金风等偏离当前项目的配色
- 是否出现大面积强渐变、重阴影、花哨色块

### 12.3 列表页检查

- 工具栏是否保持“左侧操作按钮、右侧筛选条件”的结构
- 筛选区是否遵循“下拉 -> 下拉 -> 搜索 -> 查询 -> 重置”的顺序
- 筛选区是否没有再额外包一层卡片
- 输入框和下拉高度是否统一为 `40px`
- 主操作按钮是否为蓝/灰/红三类实底按钮
- `重置` 是否仍然是白底描边按钮，没有被做成实底主按钮
- 表格表头、行高、hover、分页是否与当前列表页一致
- 行内操作是否为文字链接样式，而不是胶囊按钮
- 状态标签是否为当前项目的方圆角轻量样式，而不是过度圆润的胶囊

### 12.4 创建页 / 详情页检查

- 表单是否仍然放在白色主面板内
- 标签列与内容列是否清晰分开
- 分段区块是否仍然是浅边框、白底、适中圆角
- 上传区是否使用浅边框虚线框，而不是重色块
- 激活态是否只使用蓝色，不引入第二套强调色
- 底部操作按钮是否居中排布，主次关系明确

### 12.5 仪表盘检查

- 仪表盘是否仍然保持浅底色、白卡片、蓝色主强调
- 指标卡片是否只保留轻量视觉差异，没有脱离当前项目整体风格
- 图表区、统计区、快捷入口区是否仍然遵循统一圆角、边框和间距规则
- 是否避免把仪表盘做成“大屏风”“深色科技风”或营销页风格

### 12.6 落地检查

- 页面局部差异是否优先写在 `resources/css/pages/*.css`
- 可复用的风格差异是否已经识别并准备上收共享样式
- 如果新增了新的稳定规则，是否同步更新本文档
- 如果实现效果与本文冲突，是否优先修正页面并回写规则，而不是放任页面分叉

## 13. 维护说明

如果后续页面风格继续微调，更新本文时遵循以下原则：
- 只记录“已经在项目中稳定落地”的规则
- 不记录试验性方案
- 以最新页面效果为准，不以历史旧页为准
- 如果共享样式和本文冲突，应优先修正代码并同步修正文档

本文更新后，视为当前项目的默认页面风格基线。
