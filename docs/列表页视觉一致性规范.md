# 列表页视觉一致性规范

> 本文档聚焦后台列表页，不替代 `docs/页面风格规则.md`，而是补充更可执行的统一规则。后续新增或改造列表页时，优先遵循本文档。

## 1. 目标

当前项目已经具备基础设计 token 和通用按钮类，但列表页仍存在以下问题：

- 同类按钮在不同页面使用了不同蓝色、红色和灰色
- 列表页正文、表头、按钮文字字号不统一
- 页面 CSS 同时承担布局和视觉皮肤，导致每页都在重复定义按钮样式
- 行内操作有的使用文字链接，有的使用小按钮，有的颜色语义不一致

本规范的目标是：

- 让所有列表页在第一眼观感上属于同一套后台系统
- 让同一语义的控件始终拥有同一外观
- 让页面局部样式只负责布局差异，不再反复定义颜色和字体
- 为后续将样式逐步收口到公共 token 和公共组件提供标准

## 2. 适用范围

本文档适用于以下类型页面：

- 带筛选栏的列表页
- 带表格、分页、批量操作的管理页
- 带状态标签、行内操作、刷新按钮的表格式页面

当前仓库中，至少包括以下页面：

- `access-source`
- `gb-channel-list`
- `local-video`
- `offline-analysis`

以及后续新增的同类业务列表页。

## 3. 统一视觉原则

列表页统一遵循以下原则：

- 浅色后台，白色面板承载内容
- 信息优先，装饰次之
- 主色只用于主操作、当前状态和重点信息
- 危险色只用于删除、禁用、失败等高风险语义
- 页面之间允许布局差异，但不允许语义控件的皮肤随意变化

一句话标准：

同一种按钮、同一种表格操作、同一种状态，在任意页面都应被一眼识别为同一套规则。

## 4. 基础 Token 规范

### 4.1 字体

统一字体栈：

```css
"Source Sans Pro", "PingFang SC", "Microsoft YaHei", sans-serif
```

使用规则：

| 场景 | 字号 | 字重 | 说明 |
| --- | --- | --- | --- |
| 页面标题 / 当前面包屑 | 18px | 700 | 顶部当前页名称 |
| 面板标题 | 16px | 700 | 列表卡片标题、区块标题 |
| 筛选项标签 | 14px | 600 | 如“状态”“类型”“关键字” |
| 按钮文字 | 14px | 600 | 主按钮、次按钮、危险按钮统一 |
| 表格正文 | 14px | 400 或 500 | 优先 400，密度高时可用 500 |
| 表头 | 12px | 700 | 全部大写不是必须，但风格要稳 |
| 辅助文字 | 12px | 400 或 500 | 提示、备注、次级说明 |
| 行内操作文字 | 13px | 500 或 600 | 统一为文字链接语气 |

约束：

- 列表页中不再将正文默认写成 `13px`，正文统一提升到 `14px`
- 按钮字号不再按页面自定义为 `12px`、`13px`、`14px` 混用
- 表头固定为 `12px / 700`

### 4.2 颜色

推荐统一使用以下颜色语义，避免项目内同时出现多套“主蓝”：

```css
--color-primary: #2d83f5;
--color-primary-hover: #225fdf;
--color-primary-soft: #edf5ff;
--color-text: #273446;
--color-text-secondary: #516377;
--color-text-muted: #6f8095;
--color-border: #dbe4ef;
--color-border-strong: #c7d4e4;
--color-bg-page: #f3f6fb;
--color-bg-panel: #ffffff;
--color-bg-subtle: #f7faff;
--color-danger: #ef7270;
--color-danger-strong: #ff4d4f;
--color-danger-soft: #fff3f2;
--color-success: #56bf94;
--color-success-soft: #ebfbf4;
--color-warning: #f2a85f;
--color-warning-soft: #fff8ed;
```

约束：

- 列表页主色统一使用 `#2d83f5` 体系，不再和 `#1677ff` 混用
- 链接蓝、主按钮蓝、选中态蓝都从主色体系派生，不再各页单独挑色
- 删除相关优先使用 `#ff4d4f` 文本态或 `#ef7270` 填充态，不再出现偏粉、偏橙、偏淡红混用

### 4.3 圆角与尺寸

| 元素 | 高度 / 圆角 | 说明 |
| --- | --- | --- |
| 筛选输入框 | 40px / 6px | 列表页标准输入控件 |
| 主按钮 / 次按钮 / 危险按钮 | 40px / 6px | 顶部工具栏标准按钮 |
| 分页按钮 | 32px / 8px | 分页尺寸保持轻量 |
| 状态标签 | 24px 到 28px / 4px | 不使用超圆胶囊风格 |
| 刷新按钮 | 40px / 8px | 独立图标按钮 |
| 面板 | 10px 到 14px | 与现有后台壳层保持一致 |

约束：

- 列表页按钮不再混用 `4px`、`8px`、`999px`
- 只有标签筛选 chip 可使用胶囊形态
- 表格容器统一 `10px` 圆角

## 5. 列表页标准结构

列表页推荐结构如下：

1. 顶部面包屑和全局壳层
2. 白色面板
3. 工具栏
4. 表格容器
5. 分页区

工具栏内部再拆为两区：

- 左侧：页面级主要操作，如新增、批量删除、导入、导出
- 右侧：筛选条件、关键字搜索、查询、重置、刷新

约束：

- 筛选区与表格之间保持 `20px` 左右间距
- 不额外给筛选区再套一层厚重卡片
- 表格宽度受控于面板容器，不允许整页被表格横向撑开

## 6. 按钮规范

### 6.1 按钮分类

列表页只保留 4 类按钮语义：

| 类型 | 用途 | 样式原则 |
| --- | --- | --- |
| Primary | 当前页面最主要操作 | 主蓝底、白字 |
| Secondary | 次级但完整的按钮操作 | 白底、描边、深色字 |
| Danger | 高风险批量操作 | 红底、白字 |
| Text Link | 行内轻操作 | 透明底、彩色文字 |

不建议新增以下按钮变体：

- 每个页面自定义一套“浅灰按钮”
- 不同页面分别定义“淡红按钮”“粉红按钮”“浅蓝按钮”
- 把行内操作写成一排迷你胶囊按钮

### 6.2 顶部工具栏按钮

统一规则：

- 高度 `40px`
- 最小宽度 `96px` 到 `104px`
- 左右内边距 `16px` 到 `18px`
- 字号 `14px`
- 字重 `600`
- 主按钮和次按钮视觉权重明确，不可混淆

推荐映射：

| 文案 | 类型 |
| --- | --- |
| 查询 | Primary |
| 重置 | Secondary |
| 新增 | Primary |
| 导入 | Secondary |
| 导出 | Secondary |
| 批量删除 | Danger |
| 启用 / 发布 | Primary |
| 停用 / 终止 | Danger 或 Secondary，取决于风险级别 |

### 6.3 刷新按钮

刷新按钮统一为图标按钮：

- 尺寸 `40px * 40px`
- 白底描边
- 默认文字或图标色使用中性灰
- hover 时边框和图标色向主蓝靠拢

约束：

- 不单独发明圆形、方形、胶囊形三种刷新按钮
- 刷新按钮只负责刷新，不承载主操作语义

### 6.4 行内操作

行内操作统一使用 `Text Link` 风格：

- 默认蓝色链接表示普通操作
- 删除使用红色链接
- 字号 `13px`
- 不加背景块
- 不加描边
- hover 通过颜色加深或下划线反馈

操作数量规则：

- 单行常驻操作建议不超过 3 个
- 超过 3 个时使用“更多”收纳
- 高频操作可常驻，低频操作收进菜单

不建议：

- 在表格最后一列堆一排小按钮
- 同一列同时混用链接、胶囊按钮、实心按钮

## 7. 表格规范

### 7.1 表格容器

- 白底
- `1px` 浅边框
- `10px` 圆角
- 支持容器内横向滚动
- 超宽表格优先滚动表格容器，不放大页面主体宽度

### 7.2 表头与单元格

| 项目 | 规则 |
| --- | --- |
| 表头字号 | 12px |
| 表头字重 | 700 |
| 表头颜色 | 弱于正文，但需清晰可见 |
| 单元格字号 | 14px |
| 单元格 padding | 12px 14px |
| hover 行背景 | 使用极浅蓝灰色 |

约束：

- 表头颜色不应过浅到影响可读性
- 正文颜色不应偏灰到像“禁用态”
- 有链接或可点击内容时，只强调必要部分，不让整行都像链接

### 7.3 固定操作列

满足以下情况时，操作列应固定在右侧：

- 列数较多
- 需要横向滚动
- 行内操作频繁

规则：

- 固定列背景保持白色
- 使用轻微阴影与数据列分层
- 阴影只作为分隔，不制造悬浮卡片感

## 8. 状态标签规范

推荐统一为轻量矩形标签，不再使用高度过高的胶囊：

| 状态 | 前景色 | 背景色 | 边框色 |
| --- | --- | --- | --- |
| 成功 / 在线 / 运行中 | success | success-soft | success 辅助边框 |
| 失败 / 离线 / 已停用 | danger | danger-soft | danger 辅助边框 |
| 告警 / 待处理 | warning | warning-soft | warning 辅助边框 |
| 空闲 / 默认 | 中性灰蓝 | 浅灰蓝 | 中性描边 |

约束：

- 状态标签用于状态，不承载按钮能力
- 颜色语义不能反转，例如“失败”使用蓝色，“启用”使用灰色

## 9. 筛选区规范

筛选区的统一目标是“轻、稳、可扫读”。

规则：

- 标签字号 `14px / 600`
- 输入框、下拉框高度统一 `40px`
- 单个下拉默认宽度约 `140px`
- 搜索框默认宽度约 `220px`
- 查询与重置按钮紧邻筛选项出现
- 筛选标签与字段本身保持固定节奏，不要时大时小

推荐顺序：

1. 状态
2. 类型
3. 关键字
4. 查询
5. 重置
6. 刷新

如果页面包含页面级主操作：

- `新增`、`导入`、`批量删除` 放左侧
- `状态`、`类型`、`查询`、`重置`、`刷新` 放右侧

## 10. 分页规范

分页保持轻量，不争夺主视觉：

- 按钮高度 `32px`
- 白底描边
- 当前页使用主蓝底白字
- 禁用态通过透明度降低体现
- 不使用夸张阴影

分页信息区建议包含：

- 总数
- 当前页
- 上一页 / 下一页

## 11. 当前项目落地建议

结合现有仓库，建议按以下顺序推进：

### 第一阶段：先统一公共 token

优先收口以下内容：

- 主蓝统一到 `#2d83f5`
- 补齐按钮、字号、圆角 token
- 明确 `button`、`button-secondary`、`button-danger`、`table-action-link` 的公共定义

### 第二阶段：先改 4 个代表性列表页

建议优先处理：

1. `assets/css/pages/local-video.css`
2. `assets/css/pages/offline-analysis.css`
3. `assets/css/pages/access-source.css`
4. `assets/css/pages/gb-channel-list.css`

原因：

- 这几页同时覆盖了主按钮、危险按钮、刷新按钮、行内操作链接、表格和筛选栏
- 改完后可以沉淀出一套较稳定的列表页公共样式

### 第三阶段：页面 CSS 只保留布局差异

页面级 CSS 应优先保留：

- 列宽
- 布局结构
- 特殊业务组件
- 局部信息密度调整

页面级 CSS 不应继续负责：

- 按钮主色
- 按钮字号
- 行内操作颜色语义
- 通用刷新按钮皮肤

## 12. 验收清单

每次新增或改造列表页时，至少检查以下项目：

- 同类按钮在不同页面是否使用同一颜色和字号
- `查询 / 重置 / 新增 / 删除 / 刷新` 是否符合统一按钮语义
- 表头是否统一为 `12px / 700`
- 表格正文是否统一为 `14px`
- 行内操作是否统一为文字链接风格
- 删除操作是否始终使用危险色语义
- 刷新按钮是否保持统一尺寸和交互反馈
- 超宽表格是否仅在表格容器内横向滚动
- 状态标签是否使用统一语义色
- 页面 CSS 是否仍在写死重复的颜色值和按钮皮肤

## 13. 禁止事项

以下做法默认禁止：

- 同一个系统内长期并存两套主蓝
- 列表页正文使用过多 `12px` 或 `13px`
- 行内操作写成一排小实心按钮
- 页面随意定义新的按钮颜色名义
- 把危险操作做成低识别度的浅灰按钮
- 因为单页视觉偏好而破坏整套后台的一致性

## 14. 建议的后续文档动作

如果后续继续推进样式治理，建议补充两份文档：

- `docs/列表页组件映射表.md`
  - 记录每个页面当前使用了哪些按钮、表格、标签变体
- `docs/design-tokens.md`
  - 统一沉淀颜色、字号、间距、圆角、阴影 token

这样可以把“页面风格规则”提升为“可维护的设计系统约束”。
